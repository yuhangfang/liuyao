# 六爻算卦网站实现计划

## 一、需求与规则摘要

- **起卦**：6 组硬币，每组 3 枚；用户可点击每枚硬币在「花」「字」间切换。
- **记爻规则**（每组 3 枚统计「花」的个数）：
  - 0 花 → 老阴（X，动爻，变阳）
  - 1 花 → 少阳（—）
  - 2 花 → 少阴（- -）
  - 3 花 → 老阳（O，动爻，变阴）
- **排卦**：从下往上为初爻～六爻，得**本卦**；老阳/老阴所在爻变，得**变卦**。
- **装卦**：纳甲（干支）、卦宫、世应、六亲、六神（按起卦日干）。
- **分析**：五行生克、地支合冲刑、墓库（辰戌丑未）、明动/暗动标注，并生成**卦象判断**结论。

## 二、技术选型建议


| 层级  | 建议                                     | 说明                         |
| --- | -------------------------------------- | -------------------------- |
| 前端  | React + TypeScript（或 Vue3 + TS）        | 组件化：表单、硬币组、卦盘、分析结果         |
| 样式  | Tailwind CSS 或 CSS Modules             | 卦象、爻位需清晰排版                 |
| 状态  | React 内 useState/useReducer 或 Zustand  | 存：时间、问题、6 组硬币、排盘结果、分析结果    |
| 后端  | 可选 Node（Express/Fastify）或纯前端           | 若卦象/断辞从数据库或文件读，可放后端；否则可全前端 |
| 部署  | Vite 构建，静态托管（Vercel/Netlify）或 + 简单 API | 先做纯前端即可跑通全流程               |


若你已有技术栈（如只用 HTML/JS、或指定框架），可在实现时替换为同构逻辑。

## 三、站点结构与页面流程

- **单页即可**：同一页内从上到下分为「输入区 → 硬币区 → 排盘区 → 分析区 → 结论区」，或用步骤条（Step 1 输入 → Step 2 摇卦 → Step 3 看卦）。
- **首屏**：突出「日期时间 + 问题 + 六组硬币」，硬币为默认未选或默认一种状态，点击切换花/字。

## 四、数据结构与核心逻辑

### 4.1 输入与硬币

- **起卦参数**：`dateTime: Date`（或年月日时分开）、`question: string`。
- **每组硬币**：`[0|1, 0|1, 0|1]`（0=字，1=花），共 6 组。
- 由每组「花」的个数 → 得到 6 个爻类型：`'lao_yin'|'shao_yang'|'shao_yin'|'lao_yang'`。

### 4.2 本卦与变卦

- **本卦**：6 个爻的阴阳（老阴、少阴→阴；老阳、少阳→阳），从初爻到上爻存为数组。
- **变卦**：老阴→阳、老阳→阴，其余不变，再得 6 爻。
- 八卦由 3 爻组成：下卦 = 初爻～三爻，上卦 = 四爻～六爻；用 8 卦映射（乾兑离震巽坎艮坤）得到卦名与卦宫。

### 4.3 装卦（纳甲、世应、六亲、六神）

- **纳甲**：按京房纳甲口诀，为每个爻位配「干支」。不同卦宫、上下卦、爻位对应不同地支（及天干）。
- **世应**：按卦宫与卦的宫位（一二三四五六世、游魂、归魂）定世爻、应爻位置。
- **六亲**：以卦宫五行为「我」，各爻地支五行为「他」，生克关系 → 父母、兄弟、子孙、妻财、官鬼。
- **六神**：按起卦日干（甲乙→青龙…）从初爻往上顺排或逆排青龙、朱雀、勾陈、螣蛇、白虎、玄武。

以上需要维护：八卦纳甲表、八宫世应表、六神顺逆表等常量数据（可放在前端 `data/` 或配置文件）。

### 4.4 分析规则（生克、合冲刑、墓库、明动暗动）

- **生克**：爻地支五行与用神/世爻等之间的生克关系标注（可表格或列表）。
- **合**：六合（子丑、寅亥等）、三合（申子辰等）；**冲**：六冲（子午、丑未等）；**刑**：寅巳申、丑戌未等。
- **墓库**：地支为辰戌丑未的爻标注「墓」或「库」。
- **明动**：老阳、老阴爻即动爻，标为「明动」；**暗动**：静爻被日建/动爻冲而暗动（规则需按传统定义实现）。
- 以上结果用于生成「卦象判断」的文本：用神、吉凶倾向、关键生克与动爻影响等（可先做规则模板，再逐步细化）。

## 五、前端模块划分

1. **表单模块**：日期时间选择器、问题文本框。
2. **硬币模块**：6 组 × 3 枚，每枚可点击切换花/字；下方可显示当前组对应的爻象（—、- -、O、X）。
3. **排盘模块**：
   - 显示本卦、变卦的卦名与卦象图（六爻从下到上）。
   - 表格或列表：爻位、干支、六亲、六神、动爻标记、五行等。
4. **分析模块**：
   - 生克、合冲刑、墓库、明动/暗动的结构化标注（可逐爻或汇总）。
5. **结论模块**：一段或多段「卦象判断」文案（由分析结果 + 模板或简单规则生成）。

---

## 六、实现顺序建议

1. **项目初始化**：创建 React+Vite+TS 项目（或你选的栈），配置路由（若多页）或单页布局。
2. **数据与常量**：实现八卦、纳甲、世应、六亲、六神等静态数据与卦宫计算。
3. **首屏与输入**：首屏布局，日期时间 + 问题输入，六组硬币 UI 与花/字状态。
4. **起卦与排盘**：由 6 组硬币 → 本卦/变卦 → 装卦（纳甲、世应、六亲、六神），并展示卦象与表格。
5. **分析逻辑**：实现生克、合冲刑、墓库、明动/暗动判断与标注。
6. **断卦结论**：根据用神、动爻、生克与动爻类型生成卦象判断文案。
7. **样式与体验**：卦象图、表格可读性，移动端适配（可选）。

---

## 七、注意事项

- **传统差异**：不同流派在六神顺逆、暗动条件、用神取法上可能不同，建议先定一套规则并写成文档，便于后续扩展。
- **数据量**：纳甲、八宫、六神等表可做成 JSON/TS 常量，不依赖后端即可完成排盘与基础分析。
- **扩展**：若以后要「解卦库」「历史记录」，可再加本地存储或简单后端与数据库。

按上述顺序可实现：首屏输入与六组硬币 → 排盘 → 装卦 → 生克/合冲刑/墓库/明动暗动分析 → 卦象判断结论的全流程。
